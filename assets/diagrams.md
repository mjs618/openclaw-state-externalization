# ASCII 示意图

## 状态外化架构

```
┌─────────────────────────────────────────────────────────────┐
│                     AI Agent 上下文窗口                      │
│                    (有限, 易失, 昂贵)                       │
│                         256K tokens                          │
└───────────────────────┬─────────────────────────────────────┘
                        │
                        ▼ 状态外化
┌─────────────────────────────────────────────────────────────┐
│                      外部存储系统                            │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │ manifest.json│  │ active-task  │  │  sessions/   │      │
│  │   (全局状态)  │  │   (当前任务)  │  │ (历史记录)   │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
│                                                             │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │  task_plan   │  │   findings   │  │   progress   │      │
│  │   (任务计划)  │  │   (研究发现)  │  │   (进度日志)  │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
└─────────────────────────────────────────────────────────────┘
                        │
                        ▼ 版本控制
┌─────────────────────────────────────────────────────────────┐
│                         Git 仓库                            │
│              (持久化、协作、可追溯、可回滚)                   │
└─────────────────────────────────────────────────────────────┘
```

## Session 生命周期

```
    Session 开始
        │
        ▼
┌───────────────────┐
│ 1. 读取 manifest  │ ◄── 了解全局状态
│ 2. 读取 README    │ ◄── 了解项目概览
│ 3. 读取 active    │ ◄── 获取当前任务
│ 4. 读取 sessions  │ ◄── 了解上次状态
└───────────────────┘
        │
        ▼
   执行任务工作
        │
        ▼
┌───────────────────┐
│ 1. 更新 task_plan │
│ 2. 追加 findings  │
│ 3. 记录 progress  │
│ 4. 创建 session   │
│ 5. 更新 manifest  │
└───────────────────┘
        │
        ▼
    Session 结束
```

## 三层架构对比

```
┌────────────────────────────────────────────────────────────┐
│ Layer 1: 超大上下文模型                                     │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━                                  │
│ 容量: 100万-200万 Token                                     │
│ 成本: $$$ (昂贵)                                            │
│ 问题: Lost-in-the-middle, 上下文过长模型会"失忆"            │
│ 适用: 单次大容量处理                                        │
└────────────────────────────────────────────────────────────┘
                              ▼
┌────────────────────────────────────────────────────────────┐
│ Layer 2: Session Compaction                                 │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━                                  │
│ 机制: OpenClaw 原生，自动压缩历史                           │
│ 优点: 节省 Token                                            │
│ 缺点: 重启后仍可能丢失关键上下文                            │
│ 适用: 单次 Session 内的长期对话                             │
└────────────────────────────────────────────────────────────┘
                              ▼
┌────────────────────────────────────────────────────────────┐
│ Layer 3: 状态外化 ⭐ (本方案)                               │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━                                  │
│ 机制: 关键状态写入文件系统                                  │
│ 优点: 持久、协作、可控、跨 Session                          │
│ 成本: 极低 (文件存储)                                       │
│ 适用: 跨天、跨 Session、多人协作的长期任务                  │
└────────────────────────────────────────────────────────────┘
```

```
Agent A                    Git 仓库                   Agent B
   │                          │                          │
   │  Session 1               │                          │
   │  ├─ 读取状态             │                          │
   │  ├─ 执行任务             │                          │
   │  └─ 更新状态             │                          │
   │         │                │                          │
   │         └───────────────►│                          │
   │                    git push                          │
   │                          │◄─────────────────────────┤
   │                          │                    git pull
   │                          │                          │
   │                          │  Session 2               │
   │                          │  ├─ 读取状态 (含A的成果)  │
   │                          │  ├─ 执行任务             │
   │                          │  └─ 更新状态             │
   │                          │                          │
   │◄─────────────────────────│                          │
   │                    git pull                          │
   │                          │                          │
   │  Session 3               │                          │
   │  (继续协作)              │                          │
```

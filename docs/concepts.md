# 核心概念

## 什么是状态外化？

**状态外化** 是将 AI Agent 的工作状态从"易失的上下文窗口"转移到"持久的外部存储"的技术。

### 类比理解

| 传统方式 | 状态外化 |
|---------|---------|
| 在黑板上写字，下课擦掉 | 写在笔记本上，随时翻阅 |
| 断电后内存数据丢失 | 硬盘数据持久保存 |
| Session 结束上下文消失 | 状态文件永久保留 |

## 三层架构

```
┌─────────────────────────────────────────┐
│  Layer 1: 超大上下文模型                  │
│  容量: 100万-200万 Token                  │
│  问题: 昂贵、Lost-in-the-middle          │
├─────────────────────────────────────────┤
│  Layer 2: Session Compaction             │
│  OpenClaw 原生机制，自动压缩             │
│  问题: 重启后仍可能丢失                  │
├─────────────────────────────────────────┤
│  Layer 3: 状态外化 ⭐                    │
│  关键状态写入文件系统                    │
│  优点: 持久、协作、可控                  │
└─────────────────────────────────────────┘
```

## 核心文件

### manifest.json
机器可读的全局状态，包含：
- 项目元信息
- 当前激活任务
- 任务列表和状态
- Session 统计

### active-task.md
人可读的当前任务状态，包含：
- 任务目标和阶段
- 待办清单
- 阻塞项
- 下一步行动

### sessions/*.md
每个 Session 的完整记录，包含：
- 完成的工作
- 遇到的问题
- 关键决策
- 待续任务

## 工作流程

### Session 启动

```
1. 读取 manifest.json → 了解全局状态
2. 读取 README.md → 了解项目概览  
3. 读取 active-task.md → 获取当前任务
4. 读取最新 Session → 了解上次状态
5. 输出状态摘要 → 让用户确认
6. 开始执行工作
```

### Session 结束

```
1. 更新 task_plan.md → 标记完成项
2. 追加 findings.md → 记录新发现
3. 追加 progress.md → Session 日志
4. 创建 Session 文件 → 本次记录
5. 更新 manifest.json → 更新状态
6. git commit → 持久化变更
```

## 状态流转

```
待开始 (pending)
    ↓
进行中 (in_progress)
    ↓
审核中 (review) ─→ 退回修改
    ↓
已完成 (complete)
    ↓
已归档 (archived)
```

## 最佳实践

- **任务粒度**: 2-4 个 Session 完成一个任务
- **记录规范**: 记录"为什么"和"结果"，而非"怎么做"
- **及时提交**: 每个 Session 结束都 git commit
- **频繁同步**: 多人协作时及时 push/pull
